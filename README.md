# Браузерные тесты на Selenium

В реализации тестов используется шаблон проектирования Page Object Model

## Сущности данного шаблона

### Сценарии тестирования

> Файлы с префиксом `test_**.py`

Сценарии отражают саму бизнес-логику приложения

Вызываются только классы страниц с их методами (open, open_contacts, fill_email, fill_password, submit)

### Страницы

> Файлы в папке `pages`

Страницы сайта хранятся в соответствии с их адресами (url)

Методы класса страницы описывают какие действия может выполнить пользователь на странице

### Компоненты

> Файлы в папке `components`

Элементы страницы, которые могут быть использованы в нескольких страницах

Например: поля для ввода, список, поле для загрузки файла

## Установка

Создание виртуального окружения:

```sh
python -m venv .venv
```

Активация виртуального окружения на Linux:

```sh
source .venv/bin/activate
```

Активация виртуального окружения на Windows:

```sh
venv\Scripts\activate
```

Установка пакетов:

```sh
pip install -r requirements.txt
```

Настройка параметров окружения:

```sh
cp .env.example .env
```

Если необходимо, то внести изменения в `.env`

## Запуск

```sh
pytest -s
```
